# PLC代码审核标准

本文档详细说明PLC代码审查的完整标准和流程。

## 审核目标

1. **变量映射完整性**: 确保PLC代码中的每个变量都在IO数据库中有对应记录
2. **命名规范一致性**: 确保变量名符合模块前缀规范
3. **IO地址准确性**: 验证变量与IO地址的对应关系正确

## 审核流程

```
步骤1: 读取PLC XML源码
   └─ 解析所有变量定义
   └─ 支持数组变量（如 SFJ_KDJC[0]）
   └─ 记录变量类型和位置

步骤2: 读取Excel IO数据库
   └─ 气缸专用版: 只读"气缸"Sheet
   └─ 完整版: 读取所有Sheet
   └─ 提取变量名、IO地址、注释

步骤3: 变量匹配检查
   └─ 在Excel中查找每个PLC变量
   └─ 记录匹配成功/失败
   └─ 失败变量必须全部列出

步骤4: 命名规范检查
   └─ 验证模块前缀
   └─ 标记缺失前缀的变量
   └─ 提供修改建议

步骤5: 生成审查报告
   └─ Markdown格式
   └─ 带时间戳命名
   └─ 包含统计和问题列表

步骤6: 输出清理后源码
   └─ 去掉注释行
   └─ 去掉空行
   └─ 便于二次分析
```

## 审核标准详解

### 标准1: 变量匹配检查

**规则**: 根据代码中的注释，在Excel中查找对应的变量名称和IO地址

**要求**:
- 每个PLC变量必须在Excel IO数据库中找到对应记录
- 如果查不到，每条都必须列出，不能遗漏
- 记录变量所在的代码位置（行号）

**匹配逻辑**:
```python
# 精确匹配
plc_var_name == excel_var_name

# 支持数组下标
SFJ_KDJC[0] 匹配 Excel中的 SFJ_KDJC[0]
```

**输出格式**:
```markdown
| 序号 | 变量名 | 类型 | 位置 |
|------|--------|------|------|
| 1 | UnknownVar | BOOL | 第23行 |
```

### 标准2: 命名规范检查

**规则**: 变量名必须包含正确的模块前缀

**标准模块前缀表**:

| 模块 | 前缀 | 说明 | 示例 |
|------|------|------|------|
| 入卷 | JR_ | Join Roll | JR_DustBoxCheck |
| 下料 | XX_ | 下料 | XX_Cylinder1 |
| 气检 | QJ_ | 气检 | QJ_PressureCheck |
| 切断 | QD_ | 切断 | QD_CutterPos |
| 贴胶 | TJ_ | 贴胶 | TJ_TapeLength |
| 模切 | MQ_ | 模切 | MQ_DieCutPos |
| 出卷 | CJ_ | 出卷 | CJ_RollOut |
| 张力 | ZJ_ | 张力 | ZJ_Tension |

**问题判定**:
```python
# 缺少前缀示例
DustBoxCheck      → 问题: 缺少模块前缀
                  → 建议: 改为 JR_DustBoxCheck

Cylinder1         → 问题: 缺少模块前缀
                  → 建议: 改为 XX_Cylinder1

# 正确示例
JR_DustBoxCheck   → 通过
XX_Cylinder1      → 通过
QJ_AirPressure    → 通过
```

**自动推断逻辑**:
- 包含 "DUST", "ROLL", "JOIN" → 建议 JR_ 前缀
- 包含 "CUT", "DISCHARGE" → 建议 XX_ 前缀
- 包含 "AIR", "PRESSURE" → 建议 QJ_ 前缀
- 包含 "TAPE", "GLUE" → 建议 TJ_ 前缀

### 标准3: IO地址核对

**规则**: 验证变量与IO地址的对应关系

**检查项**:
1. 每个输入变量应有对应的I地址（如 IA2305）
2. 每个输出变量应有对应的Q地址（如 QA1001）
3. 地址格式应符合标准规范
4. 避免地址冲突

**地址格式**:
```
输入:  IAxxxx (如 IA2305)
输出:  QAxxxx (如 QA1001)
内存:  Mxxxxx (如 M100)
```

## 常见问题处理

### Q1: Excel中找不到变量，但实际存在？

**可能原因**:
1. 变量名拼写不一致（大小写、空格）
2. Excel中使用了简写或别名
3. 变量是内部变量，不需要IO映射

**处理方法**:
- 检查变量名是否完全一致
- 确认变量是否确实需要IO映射
- 如为内部变量，可在Excel中添加备注

### Q2: 变量名不符合规范，但无法修改？

**可能原因**:
1. 遗留代码，修改影响太大
2. 与外部系统对接，需要保持一致

**处理方法**:
- 在审查报告中标记为例外
- 记录原因和决策人
- 考虑逐步迁移

### Q3: 数组变量如何处理？

**规则**:
- PLC: `SFJ_KDJC[0]`, `SFJ_KDJC[1]`
- Excel: 可记录为 `SFJ_KDJC[0]` 或 `SFJ_KDJC`（数组基名）
- 审核工具支持两种格式匹配

## 审核工具使用建议

### 定期审查
- 建议每次修改PLC代码后运行审查
- 重大版本发布前必须完整审查

### 团队协作
- 电气工程师维护Excel IO数据库
- 程序员维护PLC代码
- 使用此工具交叉验证

### 历史记录
- 利用时间戳命名保留历史报告
- 便于追溯问题和改进情况

|------------|---------|
| I* | INPUT |
| Q*, O* | OUTPUT |
| M* | MEMORY |
| *FLAG* | BOOL |
| *SPEED*, *VEL* | REAL |
