# PLC代码审查工具

自动化检查PLC XML源码与Excel IO数据库的匹配性，识别命名规范问题。

## 报告格式说明

### 3. 仅在Excel中的地址
地址在Excel中有定义，但在PLC源码中未使用。

| 地址 | Excel描述 | Sheet名 | 行号 | 来源文件 |
|------|-----------|---------|------|----------|
| OD10000 | 数字量 | 数字量 | 987 | 1098 IO表-1212.xlsx |
| OD10001 | 数字量 | 数字量 | 988 | 1098 IO表-1212.xlsx |

**说明**:
- **Sheet名**: Excel中的工作表名称
- **行号**: 在Excel中的具体行号（从1开始计数）
- 显示全部条目，不截断

### 4. 仅在PLC中的地址
地址在PLC源码中有使用，但在Excel中未定义。

| 地址 | PLC变量 | 函数名 | PLC注释 |
|------|---------|--------|---------|
| IA000 | `UINT1` | FC_UintToBooL | FC_UintToBooL(UINT1:=IA300, st_15=>IA3015, ...) |
| IA0015 | `st_15` | FC_UintToBooL | FC_UintToBooL(UINT1:=IA300, st_15=>IA3015, ...) |

**说明**:
- **函数名**: 如果IO地址是函数参数，显示所属函数（如FC_UintToBooL）
- **PLC注释**: 显示代码中的行尾注释或函数调用上下文
- 自动去重：同一地址只显示一行，优先保留带函数名的条目

**常见情况**:
1. 函数参数（如FC_UintToBooL的UINT1输入参数）- 属于系统级IO，无需在Excel中定义
2. 新增IO，Excel未更新
3. 辅助/测试IO

### 5. 地址映射示例
Excel和PLC中都有定义的地址（匹配成功）。

| 地址 | Excel描述 | PLC变量 | PLC注释 |

**说明**:

## 清理后源码格式

清理规则：
1. **去掉JSON包装**: 解析 `{"CLs":[{"TXT":"..."}]}` 格式，提取TXT字段内容
2. **解码转义符**: 自动处理 `\u000d\u000a` 等Unicode转义序列
3. **删除纯注释行**: 去掉以 `//` 开头的纯注释行
4. **保留行尾注释**: 保留代码行后的 `//` 注释（用于Excel比对）
5. **删除空行**: 跳过空行
6. **保留纯代码**: 输出有效PLC代码及行尾注释

示例：
```
// 原始JSON Lines
{"CLs":[{"TXT":"YTJ_QJ_SJ1.Sr_I := IA0000; //压接前后拉带_压接前上切胶气缸缩回\r\n// 这是纯注释行\r\n"}]}

// 清理后输出
YTJ_QJ_SJ1.Sr_I := IA0000; //压接前后拉带_压接前上切胶气缸缩回
```
